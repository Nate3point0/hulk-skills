## Module 1: FUNNEL SYSTEM

**Architecture** (build in this sequence):
1. Lead Capture Page — above-fold headline, subheadline, opt-in form, single CTA
2. Thank-You / Bridge Page — confirmation + warm lead into next step
3. Sales Page — VSL or long-form copy, benefit stack, social proof, urgency close
4. Order Form — friction-minimized, trust signals, order bump offer
5. 1-Click Upsell Page — single decision, maximum desire, no re-entry of payment info
6. Downsell Page — stripped-down offer for price-objection buyers
7. Confirmation / Member Access Page — next-step instructions, community link, course access

**Copy Framework:** Problem → Agitate → Solution → Proof → Offer → Urgency → CTA

**Systeme.io specifics:**
- Use Funnel Steps (not standalone pages) so each step auto-connects
- Order bump: configured at Order Form step → "Bump" tab
- 1-click upsell: "Upsell" step type — no second payment entry required
- Tag contacts at opt-in ("Lead-[FunnelName]") and at purchase ("Customer-[ProductName]")
- Set redirect URLs at each funnel step so there are **zero dead ends**

**Automation triggers to set:**
- Opt-in → Tag added → Welcome sequence fires
- Purchase → Tag added → Course access granted + Purchase confirmation email
- Upsell accepted → Tag added → Upsell delivery email + upsell module unlocked

**Quality Checklist — Funnel System:**
- [ ] All funnel steps connected with correct redirect URLs
- [ ] Order bump displays correctly on order form
- [ ] Upsell fires after purchase (not before)
- [ ] All tags applied at correct trigger points
- [ ] Test purchase completed end-to-end
- [ ] Mobile-responsive on all pages
- [ ] Thank-you page sets expectations for next step

## Module 2: EMAIL SEQUENCE SYSTEM

**Four Required Sequences:**

### A. Welcome Sequence (5–7 emails, Days 1–7)
- Email 1 (immediate): Delivery + what to expect
- Email 2 (Day 2): Your story — who you are, why this matters
- Email 3 (Day 3): Quick win — deliver one actionable tip
- Email 4 (Day 4): Social proof — results/testimonials
- Email 5 (Day 5): Address the #1 objection
- Email 6 (Day 6): Soft pitch — introduce the core offer
- Email 7 (Day 7): Hard pitch — urgency, clear CTA

### B. Nurture Sequence (10–14 emails, ongoing)
- Value-first ratio: 80% value / 20% pitch
- Content types: tutorials, case studies, myth-busting, Q&A, behind-the-scenes
- Each email ends with a soft CTA pointing to core offer or community

### C. Sales Sequence (5–7 emails, launch or evergreen)
- Email 1: Open cart / problem reframe
- Email 2: What's inside (features → transformation)
- Email 3: FAQ / objection crusher
- Email 4: Testimonials only
- Email 5: Urgency (price increase / closing soon)
- Email 6: Last chance (closes midnight)
- Email 7 (optional): Cart closed / waitlist

### D. Re-Engagement Sequence (3 emails, 90+ day inactive)
- Email 1: "Still there?" — curiosity open
- Email 2: What they've missed — value recap
- Email 3: Decision email — stay or unsubscribe (clean the list)

**Systeme.io specifics:**
- Create each sequence as a separate **Campaign** in the Emails tab
- Use **Automation Rules** to enroll contacts in campaigns based on tags
- Set sending schedule using **Campaign Delays** (Day 0, Day 1, etc.)
- Use **A/B subject line testing** on high-volume sends
- Tag on click for behavioral segmentation (e.g., "Clicked-SalesPage-Tag")

**Quality Checklist — Email System:**
- [ ] All sequences live in Systeme.io Campaigns
- [ ] Automation rules correctly enroll based on tags
- [ ] No sequence sends duplicate emails to same contact
- [ ] Unsubscribe link functional in all emails
- [ ] All links tracked and tested
- [ ] Subject lines under 50 characters where possible
- [ ] Preview text set on every email

## Module 3: COURSE SYSTEM

**Architecture:**
- Course name follows: [Transformation Result] + [Timeframe/Method] formula
- Example: "7-Figure Funnel Blueprint: Build Your First $10K Funnel in 30 Days"

**Standard Structure:**
- Module 0: START HERE — orientation, quick win, set expectations
- Module 1–N: Core content (each module = one major outcome)
- Each lesson: delivers ONE complete, actionable result
- Lesson naming: action verb + specific outcome ("Build Your Lead Capture Page in 20 Minutes")
- Final Module: Graduation + next steps + community + upsell to next program

**Drip Scheduling (if applicable):**
- Use Systeme.io drip content to release modules on a schedule
- Prevents overwhelm, increases completion, reduces refunds
- Typical: Module unlock every 3–7 days

**Completion Logic:**
- Set up completion certificate in Course Settings
- Automate: Completion tag → "Graduate-[CourseName]" → triggers next offer email

**Student Onboarding Email Sequence (4 emails):**
- Email 1 (immediate): Welcome + course access link + how to navigate
- Email 2 (Day 2): "Start with Module 0" reminder + quick win preview
- Email 3 (Day 7): Check-in — where are you? + community invite
- Email 4 (Day 14): Halfway point encouragement + bonus resource

**Quality Checklist — Course System:**
- [ ] All modules and lessons published (not draft)
- [ ] Drip schedule confirmed if enabled
- [ ] Course access granted via automation rule on correct tag
- [ ] Completion certificate configured
- [ ] Onboarding email sequence enrolled on course enrollment trigger
- [ ] All lesson videos load correctly
- [ ] Mobile playback tested

## Module 4: COMMUNITY SYSTEM

**Setup:**
- Community name: "[Niche] [Benefit] Community" or branded name
- Welcome message: pinned post, written in founder's voice, sets tone and rules
- Rules pinned as second post: 5–7 clear community guidelines

**Engagement Structure:**
- Weekly thread prompts (schedule in advance): Win Wednesday, Question Tuesday, Share Your Work Friday
- Onboarding flow: new member intro prompt triggers on join
- Gating: access granted via tag from course purchase or membership product

**Automation:**
- Purchase → Tag → Community access granted (Systeme.io membership area or external link auto-delivered via email)
- Community join → Welcome DM or welcome email fires

**Retention Tactics:**
- Monthly live Q&A (calendar invite auto-sent via automation)
- Milestone celebrations (tag-based: "30-Day Member", "First Post")
- Leaderboard or featured member spotlights (manual or automated)

**Quality Checklist — Community System:**
- [ ] Welcome post + rules pinned at top
- [ ] Access delivery automation tested (purchase → access email arrives)
- [ ] Weekly engagement prompts scheduled 4 weeks ahead
- [ ] New member onboarding prompt active
- [ ] Community feels exclusive — no public indexing if paid

## Module 5: DIGITAL STORE SYSTEM

**Product Listing Formula:**
- Title: [Outcome] + [For Whom] + [Timeframe/Method]
- Description structure: Hook → Problem → Solution → Benefits (3–5 bullets) → Transformation → Price anchor → CTA
- Every bullet: sells the transformation, not the feature ("You'll know exactly what to say on every sales call" not "Includes 10 call scripts")

**Store Architecture:**
- Categories: group by buyer awareness level (beginner → advanced → done-for-you)
- Bundle strategy: 3 products → bundle at 2× single product price (perceived value anchor)
- Checkout: minimize fields, add trust seals, show guarantee prominently

**Automation on Purchase:**
- Tag → Product delivery email with download/access link
- Tag → Thank-you sequence (2–3 emails post-purchase)
- Tag → Upsell sequence starts 48 hours post-purchase

**Quality Checklist — Digital Store:**
- [ ] All products published with complete descriptions
- [ ] Checkout pages load under 2 seconds
- [ ] Post-purchase delivery automation tested
- [ ] Bundle pricing creates clear value anchor
- [ ] Guarantee displayed on checkout page
- [ ] All download links functional

7. **Final audit** — test every trigger, redirect, tag, email delivery, access grant

After each module: pause and confirm with user before proceeding to next.
